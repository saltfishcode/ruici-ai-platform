# AI 运行时动态配置验收清单与测试模板（V3.1）

## 1. 使用说明

- 本文用于把 `重构v3` 的设计稿转成可执行验收与测试基线。
- V3.1 的第一原则是：**先做 chat，再做 embedding，最后做 voice**。
- 验收顺序固定为：`SQL 与结构 -> 解析与策略（含故障回退） -> 生命周期与缓存 -> Chat 接入 -> 性能回归 -> Embedding -> Voice`。
- 每项必须记录证据：命令输出、日志片段、接口响应样例、测试报告链接。

## 1.1 与当前仓库对齐（包名/命名约定）

- 当前测试命名风格：`XxxTest`，使用 `@DisplayName` + `@Nested` 组织。
- 当前 AI 基础包：`com.ruici.ai.common.ai`（示例：`LlmProviderRegistryTest`）。
- 建议新增运行时配置包（与 V3 草案一致）：`com.ruici.ai.common.config.runtime`。
- 建议新增测试文件放置路径：`src/test/java/com/ruici/ai/common/config/runtime/`。
- 若是链路接入测试，按模块落到：
  - `src/test/java/com/ruici/ai/modules/simulation/service/`
  - `src/test/java/com/ruici/ai/modules/knowledgebase/service/`
  - `src/test/java/com/ruici/ai/modules/document/service/`
  - `src/test/java/com/ruici/ai/modules/voice/service/`

## 2. 发布前 DoD（Definition of Done）

### 2.1 阶段 DoD（必须全部通过）

| 阶段 | 通过标准 | 证据 |
|---|---|---|
| Phase 0: 设计收口 | 已明确只做 chat-first；已识别构造期固化 `ChatClient` 的改造点；已明确 Gateway 如何接 resolver | 设计说明 / 评审记录 |
| Phase 1: SQL 与表结构 | 新库初始化成功；老库补偿脚本可重复执行；无报错 | SQL 执行日志 + `\d+` 截图 |
| Phase 2: Resolver/Policy | 解析优先级与白名单校验符合设计；具备 `last-known-good` 兜底 | 单元测试报告 |
| Phase 3: 生命周期与 Registry 缓存 | Service 不再构造期固化动态 `ChatClient`；同 provider 不同 model 不串用；局部失效生效 | 代码检查 + 缓存测试日志 |
| Phase 4: Chat 接入 | simulation/document/knowledgebase 文本链路可按快照生效 | 3 条链路集成测试 |
| Phase 5: Embedding 接入 | 批次级快照固定；不发生 chunk 级查库；已明确是否需要重向量化 | 压测日志 + 任务日志 |
| Phase 6: Voice 接入 | 新会话读新 LLM 快照；旧会话不漂移；实时对话与异步评估使用同一会话快照 | 语音会话对照测试 + 聚焦单测 |

### 2.2 发布门禁（建议阈值）

- 功能回归通过率：`100%`（核心链路）。
- 关键自动化测试通过率：`>= 95%`（允许非关键用例临时隔离，但需记录原因）。
- 解析服务错误率：`< 0.1%`。
- DB 故障演练期间降级可用率：`>= 99%`（命中 last-known-good 或静态配置）。
- 高热接口 P95 延迟回归：相对基线劣化不超过 `10%`。
