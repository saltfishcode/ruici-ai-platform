## 4. 原文件预览能力 ✅ 已解决

- **已有实现**：后端接口（inline/attachment）、对象存储公开 URL、前端预览模态框。
- **解决方案**：前端直连对象存储 `storageUrl`，按 contentType 分发渲染策略（PDF/图片用 iframe、DOCX 用 docx-preview、Markdown 用 react-markdown、HTML 用 sandbox iframe），不可预览类型隐藏预览按钮。
- **服务器影响**：零（文件流量完全走对象存储，不经过后端）。
- **安全措施**：HTML 用 sandbox iframe 隔离脚本执行；对象存储 key 为 UUID 路径不可枚举。
