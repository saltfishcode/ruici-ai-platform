## 修复点汇总（按问题域）

### 1) 语音：用 IP 访问 WebSocket 断连

现象：用局域网 IP / 非 localhost 访问语音页面时，WebSocket 无法建立或很快断开。

主要原因：

- 后端对外返回的 `webSocketUrl` 曾硬编码 `ws://localhost:8080/...`，导致非本机访问必然失败。
- HTTP CORS / WebSocket origin 校验默认只接受 localhost（对真实 IP / 自定义端口不友好）。
- 前端在开发模式下的 fallback host 可能取到 `5173`（Vite）而不是后端端口 `8080`。

改动：

- 后端 WebSocket URL 改为基于当前请求上下文动态生成（`http -> ws`，`https -> wss`），不再硬编码 localhost。
- HTTP CORS 与 WebSocket allowed origins 改为支持 pattern（更适配本地与 LAN 调试）。
- 前端 WebSocket fallback host 优先从 `API_BASE_URL` 推导，避免 Vite 端口污染。

涉及文件：

- `src/main/java/com/ruici/ai/modules/voice/service/VoiceInterviewService.java`
- `src/main/java/com/ruici/ai/common/config/CorsConfig.java`
- `src/main/java/com/ruici/ai/common/config/CorsProperties.java`
- `src/main/java/com/ruici/ai/modules/voice/config/WebSocketConfig.java`
- `src/main/resources/application.yml`
- `frontend/src/pages/VoiceInterviewPage.tsx`

回滚建议（最小粒度）：

- 只回滚后端 URL 生成：回滚 `VoiceInterviewService.java`
- 只回滚跨域放开：回滚 `CorsConfig.java`、`CorsProperties.java`、`WebSocketConfig.java`、`application.yml`
- 只回滚前端 fallback host：回滚 `VoiceInterviewPage.tsx`

### 2) 语音：`No ToolCallback found for tool name: ...`

现象：语音 AI 回复时报错 `No ToolCallback found for tool name: test-development`。

主要原因：语音专用 `ChatClient` 被配置为启用 tools/tool-calls（`defaultToolCallbacks(...)` + `ToolCallAdvisor`），语音链路期望是“纯聊天/纯流式”，工具调用会造成不可控分支。

改动：

- 语音专用 client 改为 plain/no-tools 行为（禁用 tools/toolcall advisor），避免触发 tool callback。

涉及文件：

- `src/main/java/com/ruici/ai/common/ai/LlmProviderRegistry.java`

回滚建议（最小粒度）：

- 回滚 `LlmProviderRegistry.java` 中 voice client 的创建逻辑。

### 3) 历史页：语音评估页路由错误

现象：`/simulation/history` 进入语音评估页失败。

主要原因：前端历史页跳转使用了错误路径 `/voice-interview/${id}/evaluation`；正确路由为 `/voice/${id}/evaluation`。

改动：

- 修正历史页跳转路由。

涉及文件：

- `frontend/src/pages/InterviewHistoryPage.tsx`

回滚建议：

- 回滚 `InterviewHistoryPage.tsx` 的路由拼接。

### 4) 语音页：结束/返回跳转到不存在的 `/interviews`

现象：语音页结束、暂停或返回时跳到 `/interviews`（路由不存在）。

改动：

- 统一跳转到正确的记录页 `/simulation/history`。

涉及文件：

- `frontend/src/pages/VoiceInterviewPage.tsx`
- `frontend/src/pages/VoiceInterviewEvaluationPage.tsx`

回滚建议：

### 5) 文档列表：排序不稳定

现象：`/documents`（文档列表）默认展示顺序不稳定。

主要原因：后端曾使用 `findAll()`，没有强制按上传时间倒序排序。

改动：

- Repository 增加稳定倒序查询（上传时间倒序，id 倒序兜底）。
- Service 改为使用该查询。

涉及文件：

- `src/main/java/com/ruici/ai/modules/document/repository/ResumeRepository.java`
- `src/main/java/com/ruici/ai/modules/document/service/ResumePersistenceService.java`

回滚建议：

- 只回滚排序行为：回滚上述两个文件。

### 6) 同名文件难区分（前端选择项可读性）

现象：同名简历/文档在选择框里难以区分。

说明：后端去重基于内容 hash，与文件名无关；问题主要是 UI 展示信息不足。

改动：

- 在简历选择下拉中展示 `文件名 (上传日期)`，提升区分度。

涉及文件：

- `frontend/src/components/UnifiedInterviewModal.tsx`

回滚建议：

- 回滚 `UnifiedInterviewModal.tsx` 的 option 展示文本。

### 7) 情景模拟：选择题目数量显示成 2 倍

现象：选择题目数量 N，但页面显示/进度计算像是 2N。

主要原因：后端把主问题和追问（follow-up）都放在同一 `questions` 列表里；`totalQuestions` 曾用扁平列表长度。

改动：

- 后端 `totalQuestions` 改为只统计主问题数量。
- 前端进度显示改为按“已出现的主问题数量”计算，避免 follow-up 干扰。

涉及文件：

- `src/main/java/com/ruici/ai/modules/simulation/service/InterviewSessionService.java`
- `frontend/src/types/simulation.ts`
- `frontend/src/components/InterviewChatPanel.tsx`
- `frontend/src/pages/InterviewPage.tsx`

回滚建议：

- 若只回滚后端语义：回滚 `InterviewSessionService.java`
- 若只回滚前端进度展示：回滚 `simulation.ts`、`InterviewChatPanel.tsx`、`InterviewPage.tsx`

### 8) “基于情景模拟”：脏/无关文档缺少保护

现象：明显无关/乱码/过短的文本也会进入“基于简历出题”，造成体验很差。

改动：

回滚建议：
